37. Walk me through Lambda's git flow. 
  1. you for the repo
  2. you clone the repo
  3. you clone the repo in the desired location
  4. cd in the repo
  5. do a git checkout -b firstname-lastname, to be on your branch
  6. work on code
  7. git status to check what is the status of the work you're doing
  8. git add .(adds everything) followed by a git commit -m 'meaningful message'
  9. git push -u origin firstname-lastname
  10. on the github page where you have the forked repo, under branches, change it you your branch 
  11. on the new page, change the base repository to your branch name from the drop down menu
  12. merge the pull request.
